Auth Session: Impersonation Context Marker
DripJobs support can log into a customer account to troubleshoot. Right now there's no way to tell those sessions apart from a real customer login — making any "last login" data unreliable. This ticket adds a marker so support sessions are never counted as customer activity.

User Login Timestamp Tracking
DripJobs doesn't currently record when users log in at all — the data simply doesn't exist yet. This ticket builds the infrastructure to capture a per-user login timestamp each time someone signs in, filtered through the impersonation marker from ticket 1.
↓ prerequisite for #3

Once login data exists, this surfaces it on the dashboard — last login date and most recently active user per account. CS, Sales, Retention, and Billing get a real signal for which accounts are engaged and which need attention.
